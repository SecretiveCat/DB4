# Practical assignment 4

## У межах роботи було розроблено операційну базу даних для книжкового магазину. База даних дозволяє зберігати інформацію про авторів, книги, клієнтів, замовлення та склад замовлень.

## Структура бази даних:
База даних складається з п'яти таблиць: authors (10.000), books (500.000), customers (100.000), orders (100.000), order_items (200.000).

## Зв'язки між таблицями:
Один до багатьох (1:М): один автор - багато книг, один клієнт - багато замовлень
Багато до багатьох (М:М): одне замовлення може містити багато книг, одна книга може входити до багатьох замовлень.

## Процедура
Реалізовано процедуру pdate_book_price(), яка призначена для зміни ціни книги. А також вона повідомляє користувача, у разі відсутності книги з указаним ID

## Функція
Реалізовано функцію book_status(), яка знаходить книгу за назвою, повертає назву, ціну та визначений статус (дорога, якщо ціна більша 20 та дешева, якщо ціна менша або дорівнює 20). Якщо книга відсутня, тоді функція генерує виняток

## Користувачі БД
Створено три ролі: book_admin, який має повний доступ до бази даних, book_manager, який має право тільки переглядати та змінювати інформацію про книги, а також, book_viewer, який має право тільки перегляд таблиці книг

## ERD
ERD схема демонструє структуру бази даних, зв'язки між таблицями та типи відношень.
